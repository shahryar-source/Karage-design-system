🎨 Accessibility Guidelines

Global color rules for every platform

★ Quick Reference

Green 300 #B6F174
Highlights, accents, icons on dark backgrounds
Green 400 #3F9444
Buttons, links, interactive elements
Green 500 #134F1A
Sidebar, headers, brand dark surfaces
Red 500 #F04438
Errors, destructive actions, alerts
Gray 700 #333333
Primary text, headings, body content
Gray 400 #AFAFAF
Placeholder text, disabled, captions

1 Color Contrast

  • Always maintain at least:
  • AA 4.5:1 contrast ratio for body text and smaller UI elements.
  • AA 3:1 contrast ratio for large text (≥18pt or 14pt bold).
  • AAA 7:1 contrast ratio for body text when aiming for WCAG AAA.
  • Darker shades (/600 and /700) are safer for text on light backgrounds.
  • Lighter shades (/50–/200) should be used as backgrounds, highlights, or borders — not standalone text.

2 Role-Based Usage

Primary Green
Brand identity, primary actions, key highlights. Use /600+ for text/icons on light backgrounds.
Grays — Text Hierarchy
700–500 body text · 400–200 disabled/secondary · 200–50 backgrounds & dividers
Red — Alert / Error
Errors and destructive actions only. Avoid for decoration — it carries semantic meaning.
Supportive Colors
Charts, illustrations, accents. Never rely on these alone for critical information.

3 Avoiding Color-Only Communication

Do not use color as the only way to convey meaning.

  • Errors should include both red and an icon or message.
  • Success states should pair green with text or an icon.
  • Use icons, text labels, and tooltips to reinforce meaning.

4 Light vs. Dark Shades

/50–/100
Backgrounds, hover states, fills
/200–/300
Borders, highlights, secondary emphasis
/400–/700
Text, icons, high-contrast UI elements

5 Black & White

  • #141414 Black → Body text on light backgrounds.
  • #FFFFFF White → Text on dark backgrounds. Avoid white text on light shades like /100.

6 Subjective Colors (Orange, Blue, Zorba)

  • Use sparingly for illustrations, marketing, or accent highlights.
  • Always test contrast when used for text or key UI elements.

Final Note: Never hardcode colors directly. Use tokens (e.g., green-primary/500) so accessibility adjustments like dark mode or high contrast can scale easily.
